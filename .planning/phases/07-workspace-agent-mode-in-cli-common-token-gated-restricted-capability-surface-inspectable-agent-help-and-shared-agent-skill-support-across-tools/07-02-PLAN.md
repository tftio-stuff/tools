   - `pub enum AgentDispatch<T> { Cli(T), Printed(i32) }`
   - `pub fn parse_with_agent_surface_from<T, I>(spec: &ToolSpec, argv: I) -> Result<AgentDispatch<T>, clap::Error>`
   - `pub fn parse_with_agent_surface<T>(spec: &ToolSpec) -> Result<AgentDispatch<T>, clap::Error>`

   - build the clap tree from `T::command()`
   - inject hidden global args `--agent-help` and `--agent-skill <NAME>`

   - parse argv against the filtered tree first so ordinary `--help` and `help` output come from the pruned command tree in agent mode
   - if `--agent-help` is present, print help text via `render_agent_help(...)` and return `AgentDispatch::Printed(0)`
   - if `--agent-skill <NAME>` is present, print capability text via `render_agent_skill(...)` and return `AgentDispatch::Printed(0)`
   - when clap raises a parse error under active agent mode, suppress typo/suggestion leakage for hidden commands and hidden flags so the error output never mentions redacted entries or `Did you mean ...` hints for them
   - otherwise reconstruct typed CLI values from `ArgMatches` so downstream tools dispatch exactly as before

3. In `crates/cli-common/src/command.rs`, add shared wrappers that later rollout plans can call directly rather than open-coding the parser:
   - one helper for by-value clap entrypoints
   - one helper for by-reference entrypoints if needed
   Keep existing metadata helpers intact.

4. Add shared redaction-focused tests in `cli-common` that exercise the real parser entrypoint with:
   - ordinary `--help`
   - ordinary `help`
   - a typo of a hidden subcommand
   - a typo of a hidden long flag
   Assert that these outputs omit hidden command/flag names and never print suggestion text for redacted entries.

5. Re-export the new parse helpers from `crates/cli-common/src/lib.rs`.

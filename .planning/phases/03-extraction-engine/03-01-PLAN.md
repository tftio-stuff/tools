  <action>
Create `crates/bsky-comment-extractor/src/db.rs` with these public functions:

1. `pub fn open_db(path: &Path) -> Result<Connection, ExtractorError>`:
   - Create parent directories with `std::fs::create_dir_all`
   - Open connection with `Connection::open(path)`
   - Execute PRAGMAs: `PRAGMA foreign_keys = ON; PRAGMA journal_mode = WAL;`
   - Return connection
   - Model on `crates/silent-critic/src/db.rs` `open_db`

2. `pub fn init_db(conn: &Connection) -> Result<(), ExtractorError>`:
   - `conn.execute_batch(...)` with two CREATE TABLE IF NOT EXISTS statements:
   - Table `posts`: columns `uri TEXT PRIMARY KEY`, `author_did TEXT NOT NULL`, `text TEXT NOT NULL`, `created_at TEXT NOT NULL`, `raw_json TEXT NOT NULL`
   - Table `extractions`: columns `id INTEGER PRIMARY KEY AUTOINCREMENT`, `target_did TEXT NOT NULL`, `started_at TEXT NOT NULL`, `completed_at TEXT`, `record_count INTEGER NOT NULL DEFAULT 0`, `last_cursor TEXT`
   - Indexes: `idx_posts_author_did ON posts(author_did)`, `idx_posts_created_at ON posts(created_at)`, `idx_extractions_target_did ON extractions(target_did)`

3. `pub fn upsert_post(conn: &Connection, uri: &str, author_did: &str, text: &str, created_at: &str, raw_json: &str) -> Result<(), ExtractorError>`:
   - `INSERT OR REPLACE INTO posts (uri, author_did, text, created_at, raw_json) VALUES (?1, ?2, ?3, ?4, ?5)`
   - Uses `rusqlite::params!`

4. `pub fn db_has_uri(conn: &Connection, uri: &str) -> Result<bool, ExtractorError>`:
   - `SELECT 1 FROM posts WHERE uri = ?1 LIMIT 1`
   - Returns `true` if row found, `false` otherwise
   - Use `conn.query_row` with `optional()` pattern

5. `pub fn save_cursor(conn: &Connection, target_did: &str, cursor: Option<&str>) -> Result<(), ExtractorError>`:
   - If extraction row exists for `target_did` (most recent by `id DESC`), update its `last_cursor` and `record_count` (increment)
   - If no row exists, INSERT a new extraction row with `started_at = chrono::Utc::now().to_rfc3339()`, `last_cursor = cursor`, `record_count = 0`
   - Use `INSERT OR REPLACE` or a two-step SELECT + UPDATE/INSERT

6. `pub fn load_resume_cursor(conn: &Connection, target_did: &str) -> Result<Option<String>, ExtractorError>`:
   - `SELECT last_cursor FROM extractions WHERE target_did = ?1 AND completed_at IS NULL ORDER BY id DESC LIMIT 1`
   - Returns `Option<String>` (None if no incomplete extraction or cursor is null)

7. `pub fn complete_extraction(conn: &Connection, target_did: &str, record_count: u64) -> Result<(), ExtractorError>`:
   - Update the most recent extraction row for `target_did`: set `completed_at = chrono::Utc::now().to_rfc3339()`, `record_count = ?`

Every public function and the module itself has a doc comment. All error conversions go through `ExtractorError::Db` via the `From<rusqlite::Error>` impl.

Add `#[cfg(test)] mod tests` at the bottom with tests per the behavior block. All tests use `Connection::open_in_memory()` + `init_db(&conn)` for setup.

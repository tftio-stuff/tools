Create a reusable metadata-command layer with these exact API names and fields:

1. Add `crates/cli-common/src/app.rs` defining:
   - `pub struct ToolSpec { pub bin_name: &'static str, pub display_name: &'static str, pub version: &'static str, pub license: LicenseType, pub repo: RepoInfo, pub supports_json: bool, pub supports_doctor: bool, pub supports_update: bool }`
   - `impl ToolSpec { pub const fn new(...) -> Self }`

2. Add `crates/cli-common/src/command.rs` defining:
   - `pub enum StandardCommand { Version { json: bool }, License, Completions { shell: clap_complete::Shell }, Doctor, Update { version: Option<String>, force: bool, install_dir: Option<std::path::PathBuf> } }`
   - `pub fn run_standard_command<T, D>(spec: &ToolSpec, command: &StandardCommand, doctor: Option<&D>) -> i32 where T: clap::CommandFactory, D: DoctorChecks`
   - exact dispatcher behavior:
     - `Version { json: true }` prints `{"version":"<version>"}`
     - `Version { json: false }` prints `<bin_name> <version>`
     - `License` prints `display_license(spec.bin_name, spec.license)`
     - `Completions` delegates to `generate_completions::<T>(shell)`
     - `Doctor` runs `run_doctor(doctor.expect(...))`
     - `Update` delegates to `run_update(&spec.repo, spec.version, version.as_deref(), force, install_dir.as_deref())`

3. Update `crates/cli-common/src/lib.rs` to `pub mod app; pub mod command;` and re-export `ToolSpec` and `StandardCommand`.

4. Update `crates/cli-common/src/doctor.rs` to add `pub fn run_doctor_with_header<T: DoctorChecks>(tool: &T, header: &str) -> i32` and keep `run_doctor()` delegating to it.

5. Add unit tests in `app.rs`, `command.rs`, and `doctor.rs` that assert:
   - `ToolSpec::new` preserves all exact field values
   - `StandardCommand::Version { json: true }` produces a string containing `"version"`
   - `StandardCommand::License` path uses `display_license`
   - `run_doctor_with_header` prints the supplied header string and returns `1` when a check fails

Implement the shared rendering layer with exact compatibility targets:

1. In `crates/cli-common/Cargo.toml`, add these workspace dependencies:
   - `serde_json.workspace = true`
   - `indicatif.workspace = true`

2. Add `crates/cli-common/src/json.rs` with exactly these functions:
   - `pub fn ok_response(command: &str, data: serde_json::Value) -> serde_json::Value`
   - `pub fn err_response(command: &str, code: &str, message: &str, details: serde_json::Value) -> serde_json::Value`
   Both must preserve the exact object shape currently used by `todoer` and `silent-critic`.

3. Add `crates/cli-common/src/error.rs` with:
   - `pub fn print_error(command: &str, json: bool, message: &str) -> i32`
   - when `json == true`, print `err_response(command, "ERROR", message, json!({}))` to stdout and return `1`
   - when `json == false`, print `error: {message}` to stderr and return `1`

4. Add `crates/cli-common/src/progress.rs` with:
   - `pub fn make_spinner(enabled: bool, message: &str) -> Option<indicatif::ProgressBar>`
   - return `None` when `enabled == false` or stderr is not a TTY
   - otherwise create a spinner on stderr using the exact template `"{spinner:.green} {msg}"`
   - call `enable_steady_tick(std::time::Duration::from_millis(120))`

5. Update `crates/cli-common/src/output.rs` to add `pub fn stderr_is_tty() -> bool` and keep `is_tty()` for stdout.

6. Update `crates/cli-common/src/lib.rs` to export `json`, `error`, and `progress` modules and re-export `ok_response`, `err_response`, and `make_spinner`.

7. Add tests proving:
   - `ok_response("list", json!({"x":1}))` contains `ok=true`, `command="list"`, `data.x=1`
   - `err_response("list", "ERROR", "bad", json!({}))` contains `ok=false` and nested `error.code`
   - `make_spinner(false, "x")` returns `None`
   - `stderr_is_tty()` returns a bool without panicking
  </action>
